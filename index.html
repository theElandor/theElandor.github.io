1.2 Sfocatura di un immagine (17/03/22)

Vengono di seguito riportati i passi fondamentali:

  • Applico una matrice data (per effettuare la sfocatura) all' array ottenuto dalla matrice originale incolonnando tutti gli elementi
  • Stampo l'immagine e mi rendo conto di aver effettuato la sfocatura;
  • Provo a risolvere il sistema \(Hx=b\) per vedere se dall'immagine sfocata riesco a ottenere l'immagine originale(ossia x);
  • Mi rendo conto che l'immagine ottenuta è abbastanza fedele all'originale;
  • Cerco di capire se il problema è mal-condizionato / ben-condizionato: vedo se applicando una perturbazione ai dati di partenza (ossia all'immagine sfocata) riesco lo stesso ad ottenere una soluzione fedele all'immagine originale.
  • Stampando la soluzione del sistema partendo dai dati perturbati mi rendo conto che l'immagine è totalmente distorta: basta un piccolo cambiamento per rendere il sistema "quasi irrisolvibile."
  • Il problema è mal condizionato.
[n,m] = size(obj);
x = obj(:);
b = H*x; %applico la sfocatura
b_immagine=reshape(b,n,m);
imshow(b_immagine,[]); % immagine originale --> sfocatura
sol1 = H\b;
sol1_immagine = reshape(sol1,n,m); %trasformo il vettore risultante in una colonna
figure
imshow(sol1_immagine,[]); % sfocatura --> immagine originale
Deltab = randn(n*m,1); %applico un minimo errore
b2 = b +Deltab*1e-8;
b2_immagine = reshape(b2,n,m);
figure
imshow(b2_immagine, []);  %immagine sfocata con rumore applicato

sol2 = H\b2; % risolvo il sistema Hx=b2, ignorando il rumore
sol2_immagine = reshape(sol2,n,m);
figure
imshow(sol2_immagine, []); immagine con rumore --> immagine originale

1.3 Qualche iterazione del metodo di gauss (24/03/22)

big fat warning: i moltiplicatori e le matrici L sono state calcolate a mano, poi sono state riportate su matlab.

A=[1,1,1;1,1.0001,2;1,2,2];
b=[1;2;1];
format long;
res = A\b;

\(A_2 = L_1A_1\) 

L1=[1,0,0;-1,1,0;-1,0,1];
A2=L1*A

Trovo \(A_2\) correttamente! Ora cerco \(L_2\) per continuare con l'algoritmo 

L2=[1 0 0; 0 1 0; 0 -10000 1]
A3=L2*A2
U=A3

Ora calcolo la L finale, che contiene i moltiplicatori cambiati di segno rispetto alle L precedenti.

L=[1 0 0; 1 1 0; 1 10000 1]

Ora scrivo il sistema per trovare la soluzione x del sistema iniziale:
\(Ax = B\)
\(LUx=b\)

\begin{cases} Ly = B \\ Ux = y \end{cases}

Ora calcolo x usando i seguenti comandi: 

y=L\b;
x=U\y;

La soluzione usando il metodo di gauss è la seguente:
\(x=0.999999999999281,-1.000100010000281,1.000100010001000\)
La soluzione iniziale "corretta" era la seguente:
\(res=1.000000000000000,-1.000100010001000,1.000100010001000\)
Si nota quindi un certo errore di approssimazione dovuto alle operazioni floating point.

1.4 Esercitazoine fattorizzazione LU 29/03/22

big fat warning: il nome del file deve essere uguale al nome della funzione.

1.4.1 Metodo senza pivoting

  1. Codice sorgente
    function [L,U] = my_lu_dec(A)
n = length(A);
for k = 1:n-1
    if(abs(A(k,k))<eps)
        error("\n fattorizzazione non eseguibile \n");
    end
    A(k+1:n,k) = A(k+1:n,k) / A(k,k);
    A(k+1:n,k+1:n) = A(k+1:n,k+1:n) - A(k+1:n,k)*A(k,k+1:n);
end
L = tril(A,-1);
L = L+eye(n);
U = triu(A);
  2. Commenti
    • Restituisce il triangolo inferiore della matrice A.
    L = tril(A,-1);
    • Per lavorare su dei floating point, è meglio non usare zero, quindi uso eps che è la precisione di macchina. Verifico che i ppivot sulla diagonale principale siano \(!= 0\)
    if(abs(A(k,k))<eps)
    • Calcolo i moltiplicatori: prendo i vettori della sottodiagonale e lo divido per l'elemento della diagonale.
    A(k+1:n,k) = A(k+1:n,k) / A(k,k);
    • Uso i moltiplicatori per annullare gli elementi sotto la diagonale principale. In realtà non memorizzo gli zeri, tengo quello spazio per memorizzare direttamente la matrice L!
    A(k+1:n,k+1:n) = A(k+1:n,k+1:n) - A(k+1:n,k)*A(k,k+1:n);

    Questa è la matrice che ottengo dopo la fine del ciclo.

    \begin{matrix} u_{11} & u_{12} & u_{13} \\ l_{21} & u_{22} & u_{23} \\ l_{31} & l_{32} & u_{33} \end{matrix}
    • Con le ultime 3 righe di codice:
      1. Estraggo la matrice inferiore L escludendo la diagonale principale.
      2. Aggiungo degli uni sulla diagonale principale della matrice L.
      3. Estraggo la matrice U.
    • Usando le operazioni in forma matriciale invece che usando un ciclo for, matlab ottimizza le operazioni e rende tutto più efficiente.
    L = tril(A,-1);
L = L+eye(n);
U = triu(A);

1.4.2 Metodo LU con pivoting

\(LU = PA\)

  • All'inizio la matrice P è la matrice identità.
  • Il pivot deve essere \(!=0\) e deve essere maggiore degli altri elementi della colonna: in questo modo tutti i moltiplicatori saranno minori di 1, e avrò meno problemi a livello di precisione di macchina.
  • In questo modo ho che tutti gli \(m_{ik}<1\)

Di seguito viene riportata la soluzione (nostra, a quanto pare funzionante).

  1. Codice sorgente
    function [L,U,P] = my_lu_dec(A)
n=length(A);
P = eye(n);
for k=1:n-1
    [m,i] = max(A(k:n,k));
    if(m == 0)
        error("\n fattorizzazione non eseguibile \n");
    end
    if(A(k:1,k:1) ~= m)
        I = eye(n);
        I([k i],:) = I([i k], :);
        P = I*P;
        A = I*A;
    end
    A(k+1:n,k)=A(k+1:n,k)/A(k,k);
    A(k+1:n,k+1:n)=A(k+1:n,k+1:n) - A(k+1:n,k) * A(k,k+1:n);
end
L=tril(A,-1);
L=L+eye(n);
U=triu(A);
  2. Commenti
    1. Creo una matrice di permutazione \(P\) elementare per partire
    2. Ogni volta cerco il massimo della colonna in cui sono: controllo che non sia \(=0\), in tal caso, la fattorizzazione non è eseguibile.
    3. Scambio la riga i con la riga \(k\) per portare in pivot l'elemento che ho trovato.
    4. Mi tengo in memoria tutti gli scambi effettuati moltiplicando la vecchia \(P\) per la nuova \(P\).
    5. Applico la matrice \(P\) alla matrice \(A\), e di conseguenza anche alla matrice \(L\).
